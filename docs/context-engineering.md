# コンテキストエンジニアリング運用ガイド

> CLAUDE.md肥大はコスト20%増、成功率低下(ETH Zurich 2026)
> 一方、正しく精選された常駐コンテキストは100%の通過率を達成(Vercel 2026)
> 検証ループが品質を2〜3倍にする(Boris Cherny実証)
> このガイドはCLAUDE.md自体をメンテナンスする際の判断基準

## 精選の原則

- 書くのは「新しいチームメンバーが初日に間違えること」だけ
- AIが自力で推論できること (SOLID定義、RED/GREEN/REFACTOR手順等)は書かない
- 効果のないルールは積極的に削除。「削除したらミスするか？」でテスト

## 運用

- ミス発生 → 修正指示 →「CLAUDE.mdを更新して」→ 効果を測定
- 200行を超えたら剪定。低頻度の指示はskillsに移動
- PRレビュー時にCLAUDE.mdの改善も検討

## アンチパターン

- LLMにCLAUDE.mdを自動生成させる → 成功率低下
- ディレクトリ構造を列挙する → AIは自分で探索可能
- コードスニペットを埋め込む → 陳腐化
- 「MUST」「NEVER」の多用 → ファイルが長いと強調効果消失
