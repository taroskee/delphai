# Wardley Map 駆動の要件定義

> 「何を自作し、何を既製品で済ませるか」の判断を勘ではなく進化段階で行うため

## このプロジェクトでの判断基準

- 新機能・技術選定の前に、対象コンポーネントの進化段階を判定する
- Genesis/Custom → 自作し、不確実性を受け入れ、変更容易性を重視
- Product → 既存ライブラリ/フレームワーク、ロックイン回避にアダプター層
- Commodity → マネージドサービス/SaaS、自作しない(認証、メール送信、ストレージ等)
- 現在Genesisでも将来Commodityになるため、アダプター層で依存を抽象化しておく

## チェックリスト

- [ ] ユーザーニーズは明確か？

- [ ] Commodityを自作していないか？
- [ ] Genesisの不確実性を設計に織り込んだか？
